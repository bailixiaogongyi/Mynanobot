# 主题笔记技能

## 功能

- 在 `topics/` 目录下创建主题笔记
- 支持按主题组织内容，便于深度记录
- 依据临时笔记内容进行整合和加工
- 形成结构清晰、逻辑合理、内容完善的笔记

## 目录位置

```
workspace/
└── topics/            # 主题笔记目录
    ├── 学习笔记.md
    ├── 技术方案.md
    └── 课题研究.md
```

## 使用方式

### 创建主题笔记

当用户要求创建主题笔记时，使用 `write_file` 工具在 `topics/` 目录下创建文件。

示例：

```
write_file("topics/技术方案.md", "# 技术方案\n\n## 背景\n\n## 方案详情\n\n## 注意事项\n")
```

### 从临时笔记整合内容

当用户要求将临时笔记整合为主题笔记时：

1. 读取相关的临时笔记文件
2. 分析和整理内容
3. 按主题结构重新组织
4. 创建或更新主题笔记
5. 删除已处理的临时笔记

示例流程：

```
# 1. 读取临时笔记
read_file("pending/想法记录.md")

# 2. 整合内容，创建主题笔记
write_file("topics/产品设计.md", "# 产品设计\n\n## 核心想法\n\n...\n\n## 详细设计\n\n...")

# 3. 删除已处理的临时笔记
delete_file("pending/想法记录.md")
```

### 追加内容

使用 `edit_file` 工具追加内容到主题笔记。

示例：

```
edit_file("topics/技术方案.md", old_content="## 注意事项\n", new_content="## 注意事项\n- 新增注意事项\n")
```

### 查看主题笔记

使用 `read_file` 工具读取主题笔记。

示例：

```
read_file("topics/技术方案.md")
```

### 搜索主题内容

使用 `note_search` 工具搜索主题笔记内容。

示例：

```
note_search("技术方案", top_k=10)
```

## 主题笔记结构建议

```markdown
# {主题名称}

## 概述

简要描述主题的核心内容。

## 背景

主题产生的背景和上下文。

## 详细内容

### 子主题1

### 子主题2

## 相关资源

- 链接1
- 链接2

## 更新记录

- YYYY-MM-DD: 初始创建
- YYYY-MM-DD: 更新内容
```

## 与其他笔记的区别

| 类型     | 用途               | 位置                   |
| -------- | ------------------ | ---------------------- |
| 每日笔记 | 记录当天事件、任务 | `memory/YYYY-MM-DD.md` |
| 临时笔记 | 快速记录临时想法   | `pending/{主题}.md`    |
| 主题笔记 | 深度记录特定主题   | `topics/{主题}.md`     |
| 项目笔记 | 项目相关信息       | `projects/{项目名}.md` |

## 注意事项

- 主题笔记适合长期积累的内容
- 使用清晰的文件命名，便于检索
- 定期整理和更新主题笔记
- 从临时笔记整合时，要进行内容加工，不是简单复制
- 保持结构清晰，便于后续查阅和扩展
