# OCR光学字符识别技能

## 核心原则

- 操作前先确认图片/文件路径是否正确
- 告知用户图像清晰度会影响识别准确率
- 对于模糊或低质量的图片，如实告知识别可能不准确
- 批量处理时，注意处理时间可能较长
- 返回结果要清晰说明识别了多少内容

## 触发条件

- 用户提到"识别文字"、"OCR"、"图片转文字"
- 用户提到"截图识别"、"照片识别"
- 用户提到"名片识别"、"发票识别"
- 用户提到"PDF OCR"、"扫描件识别"
- 用户提到"批量识别"、"批量OCR"
- 用户发送一张图片并要求识别文字

## 执行步骤

### 场景1：识别单张图片文字

1. 确认用户提供的图片路径
2. 使用 `ocr_recognize` 进行识别
3. 如果需要详细信息（包含置信度和位置），使用 `detail: true`
4. 返回识别的文字内容

### 场景2：批量识别图片

1. 确认图片所在目录
2. 使用 `ocr_batch` 批量处理目录中的所有图片
3. 可选择输出JSON文件保存结果
4. 返回各图片的识别结果

### 场景3：PDF OCR识别

1. 确认PDF文件路径
2. 如果需要特定页面，使用 `pages` 参数
3. 使用 `ocr_pdf` 进行OCR识别
4. 返回各页的识别结果

## 注意事项/安全守则

- 只处理用户指定的图片/PDF文件，不访问其他文件
- 不执行任何shell命令或安装操作
- 确保输出目录有写入权限
- 告知用户识别准确率受图片质量影响
- 对于严重模糊或损坏的图片，如实告知可能无法识别
- 不处理涉及隐私或敏感内容的图片
