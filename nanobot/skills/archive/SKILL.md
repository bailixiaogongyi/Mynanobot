# 归档技能

## 功能

- 将超过保留期限的每日笔记从 `memory/` 移动到 `daily/` 目录
- 按年月组织归档文件
- 保持 `memory/` 目录整洁，控制上下文大小

## 归档规则

| 文件类型 | 保留期限 | 源目录    | 归档目录         |
| -------- | -------- | --------- | ---------------- |
| 每日笔记 | 7 天     | `memory/` | `daily/YYYY/MM/` |

## 使用方式

### 手动归档

使用文件操作工具移动超过 7 天的每日笔记到归档目录：

1. 列出 `memory/` 目录下的所有 `.md` 文件
2. 识别文件名格式为 `YYYY-MM-DD.md` 的文件
3. 计算文件日期是否超过 7 天
4. 移动到 `daily/YYYY/MM/YYYY-MM-DD.md`

示例步骤：

```
# 1. 列出 memory 目录
list_dir("memory")

# 2. 读取源文件
read_file("memory/2026-02-20.md")

# 3. 创建目标目录（如不存在）
# 目标路径: daily/2026/02/2026-02-20.md

# 4. 写入目标文件
write_file("daily/2026/02/2026-02-20.md", content)

# 5. 删除源文件
delete_file("memory/2026-02-20.md")
```

### 自动归档

建议配置 cron 任务每日执行归档操作：

```
# 每天凌晨 2 点执行归档
0 2 * * * 归档超过 7 天的每日笔记
```

## 目录结构

```
workspace/
├── memory/           # 最近 7 天的每日笔记
│   ├── MEMORY.md     # 长期记忆
│   ├── HISTORY.md    # 历史日志
│   ├── 2026-03-01.md
│   ├── 2026-03-02.md
│   └── ...
└── daily/            # 归档笔记
    └── 2026/
        └── 02/
            ├── 2026-02-01.md
            ├── 2026-02-02.md
            └── ...
```

## Token 优化

归档后，`memory/` 目录只保留最近 7 天的笔记，大幅减少日常使用的上下文大小。

## 注意事项

- 归档前确保目标目录存在
- 归档后可通过 `note_search` 工具检索历史笔记
- 建议在低峰期执行归档操作
- 不要归档 `MEMORY.md` 和 `HISTORY.md` 文件
