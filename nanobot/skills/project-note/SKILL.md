# 项目笔记技能

## 功能

- 在 `projects/` 目录下创建项目笔记文件
- 提供标准的项目文档结构模板
- 从每日笔记中提取项目相关信息
- 智能整合项目信息，而非简单追加

## 目录位置

```
workspace/
└── projects/          # 项目笔记目录
    ├── 项目A.md
    ├── 项目B.md
    └── ...
```

## 使用方式

### 创建项目笔记

当用户要求创建项目时，使用 `write_file` 工具在 `projects/` 目录下创建文件。

示例：

```
write_file("projects/智能助手项目.md", "# 智能助手项目\n\n## 项目背景\n\n...\n\n## 项目目标\n\n...\n")
```

### 项目笔记默认结构

创建项目笔记时，应包含以下结构：

```markdown
# {项目名称}

## 项目背景

描述项目的背景和起因。

## 项目目标

- 目标1
- 目标2
- 目标3

## 项目内容

### 功能范围

### 技术方案

## 重要信息

### 关键决策

| 日期       | 决策内容 | 原因  |
| ---------- | -------- | ----- |
| YYYY-MM-DD | 决策1    | 原因1 |

### 风险事项

- 风险1
- 风险2

## 涉及人员

| 角色     | 姓名 | 职责     |
| -------- | ---- | -------- |
| 项目经理 | XXX  | 整体协调 |
| 开发     | XXX  | 核心开发 |

## 项目进展

### YYYY-MM-DD

- 完成事项1
- 完成事项2

### YYYY-MM-DD

- 完成事项3
- 进行中事项1

## 待办事项

- [ ] 待办1
- [ ] 待办2

## 相关资源

- 文档链接1
- 文档链接2
```

### 从每日笔记提取项目信息

当用户要求更新项目进展或提取项目信息时：

1. 列出 `memory/` 目录下的每日笔记文件
2. 读取相关的每日笔记内容
3. 识别与该项目相关的信息
4. 读取现有项目笔记内容
5. 智能整合新信息到项目笔记中
6. 避免重复，保持结构清晰

示例流程：

```
# 1. 列出每日笔记
list_dir("memory")

# 2. 读取相关日期的笔记
read_file("memory/2026-03-01.md")
read_file("memory/2026-03-02.md")

# 3. 读取现有项目笔记
read_file("projects/智能助手项目.md")

# 4. 整合信息，更新项目笔记
edit_file("projects/智能助手项目.md",
    old_content="## 项目进展\n\n### 2026-03-01\n",
    new_content="## 项目进展\n\n### 2026-03-02\n- 新进展1\n- 新进展2\n\n### 2026-03-01\n")
```

### 更新项目进展

使用 `edit_file` 工具追加进展记录。

示例：

```
edit_file("projects/智能助手项目.md",
    old_content="## 项目进展\n",
    new_content="## 项目进展\n\n### 2026-03-02\n- 完成功能A开发\n- 开始功能B设计\n")
```

### 查看项目笔记

使用 `read_file` 工具读取项目笔记。

示例：

```
read_file("projects/智能助手项目.md")
```

### 列出所有项目

使用 `list_dir` 工具列出 `projects/` 目录。

示例：

```
list_dir("projects")
```

## 信息整合原则

从每日笔记提取信息时，应遵循以下原则：

1. **去重**：避免重复记录相同内容
2. **归类**：将信息放入正确的章节
3. **精简**：提取关键信息，去除冗余
4. **时效**：进展记录按日期倒序排列
5. **完整**：确保重要信息不遗漏

## 与其他笔记的区别

| 类型     | 用途               | 位置                   |
| -------- | ------------------ | ---------------------- |
| 每日笔记 | 记录当天事件、任务 | `memory/YYYY-MM-DD.md` |
| 临时笔记 | 快速记录临时想法   | `pending/{主题}.md`    |
| 主题笔记 | 深度记录特定主题   | `topics/{主题}.md`     |
| 项目笔记 | 项目相关信息       | `projects/{项目名}.md` |

## 注意事项

- 项目笔记是项目信息的核心载体
- 定期从每日笔记中提取和整合项目信息
- 保持项目笔记的结构清晰和内容完整
- 重要决策和风险事项要及时记录
- 进展记录要简洁明了
